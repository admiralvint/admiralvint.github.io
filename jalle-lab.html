Case Study

Jälle Lab — Monitoring Platform

A full-stack IoT monitoring system built for a battery recycling R&D facility in Tallinn — from physical sensors on the lab floor to a purpose-built web app for lab operators.

Context

The lab and the problem

Jälle Technologies runs an active R&D facility focused on battery recycling. Their experiments are temperature-sensitive and continuous — environmental conditions need to be tracked reliably across multiple zones throughout each run.

Before this system, tracking was done manually. That meant no easy way for lab staff to review a completed experiment or configure conditions for the next one.

What was built

The full stack

Custom thermocouple hardware — from concept to bench

The lab needed high-temperature measurement that consumer sensors can't cover. The solution was custom-built: ESP32 nodes paired with K-type thermocouples and a MAX31856 amplifier, wired and assembled in-house for each measurement point.

Getting clean readings in the lab took work — industrial equipment nearby caused erratic multi-degree EMI spikes. Fixed through careful board placement and a firmware spike-rejection filter that requires consecutive-sample confirmation before accepting a temperature shift.

Enclosures designed in OpenSCAD, printed on a Bambu Lab P2S. The amplifier ended up in a separate magnetically-mounted box — co-locating it with the ESP32 made the EMI worse.

Industrial integration and MQTT backbone

Lumel RE92 industrial temperature controllers — already in use for process control — are integrated over Modbus TCP via Node-RED, bringing their readings into the same pipeline without replacing existing hardware. A Mosquitto broker acts as the common messaging backbone; all nodes publish to a unified topic structure regardless of origin.

Data pipeline

Node-RED handles protocol bridging and ingestion — normalising readings from MQTT and Modbus TCP before writing them into InfluxDB for time-series storage. Grafana sits on top for ops-level visibility: full history, raw query access, and alerting.

Infrastructure

The full stack runs on a dedicated lab server, containerised with Docker. Everything operates on-premises within the lab network — no cloud dependency for the data path.

Operator dashboard (React / TypeScript)

The centrepiece of the project: a custom web application built for lab staff who don't have a Grafana background. It provides a clean interface for day-to-day lab work — creating and annotating experiment runs, configuring per-experiment thresholds, and reviewing sensor data from a completed run without needing to write queries.

The dashboard talks directly to InfluxDB and exposes only what's relevant to the operator's workflow, keeping the interface focused and approachable for non-technical users.

Result

Outcome

Manual data logging is gone. Environmental data is captured automatically and continuously for every experiment, with a clear audit trail tied to the experiment record created by the operator before the run starts.

Lab staff now have a tool that fits their workflow — not a repurposed ops dashboard. The operations team retains full visibility through Grafana when they need it.
